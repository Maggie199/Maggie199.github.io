troubleshooting

performance gap

People make plans everyday, but the gap between expected and the actual performance is rarely closed.

GitHub msg vs project progress

GitHub commit messages tell us a lot about software dev project progress, which is important for project managers to guarantee the overall team efficiency

The Challenges

Tracks developer's coding performance by parsing their commit messages
Breaks project into milestones/sections for developers to track their own progress
Shows the progress of whole project to PMs by data visualization
Calculates difference between expected & actual performance on a daily base
1

Feasibility Study

Brainstormed as a team and communicated with our Client - the Small Data Lab @ Cornell Tech in NYC and narrowed down their requirement from "an application that tracks the time it takes users to complete real world tasks" to "an web app to improve productivity, collaborative work and project management", and finally targeted it at GitHub users. Demonstrated that the proposed system is technically feasible, recommended a process based on iterative refinement, and provided an outline plan.

Functional Requirements:

  • - Authorization of User
  • - Project creation and tracking
  • - Data collection with Github API
  • - Data Analysis
  • - Data Visualization
  • - Distinct interfaces for developers and PMs

Non-functional Requirements:

  • - Portability
  • - Extensibility
  • - Documentation
  • - Maintenance
2

Requirement Analysis

Created use case diagram, flow charts and database Entity-­‐Relation(ER) diagram to analyze and specify requirements of the system.

3

System Design

Designed the System Architecture of Model-­View-­Controller, to meet the requirements defined in step2. As shown below, the controller section receives user input events and communicate with Github for data collection; the view section interprets data and present it to users; while the model section writes/reads data from database and connects the other two sections.

4

Initial UI Design

Created a rapid prototype of the initial design using Bootstrap with the following functionalities:

  • - A landing page to illustrate the project ideas clearly to users
  • - Log-in with existing Github account
  • - A dashboard to show all projects
  • - Add a new project by filling out a form
  • - A project overview page for PMs to compare expected and actual progress
  • - A project detail page for developers to track task progress
5

1st Usability Testing

Conducted the first round usability testing on five participants focusing on three elements of Usability: Effectiveness, Efficiency, Satisfaction, and got the following insights:

Dashboard page:

  • - It's not clear what the percentage means
  • - Unable to tell a user's role in a project
  • - The color schema on dashboard makes less sense

Developer's view page:

  • - Only one ongoing task could be displayed
  • - Low color contrast

PM's view page:

  • - It's not clear that the height of bars means percentages
  • - Not intuitive about how much work needs to be done

Set-up form page:

  • - Unable to add multiple sections after the first one
  • - Unable to set up milestones
  • - The way to paste Github url and import projects is inefficient
6

Revised System Design

Improved system design according to advices from professor, and created sequence diagram to show how messages are dispatched among different part of the system in time ordering.

Built class diagram with extra columns in each table of our database to meet the extensibility requirement.

7

Revised UI Design & Development

Improved initial UI design based on the key findings from the 1st usability testing. Implemented those changes with HTML, CSS & JavaScript and used JSON & Ajax to read data from database. Integrated frontend and backend development with the framework of Django.

9

3rd Iteration improvements

Improved frontend design & development based on the key findings from the 2nd usability testing and client's feedback. Designed an error page to improve the completeness and security of the system and prevent malicious users from accessing pages without authorization by modifying urls. Implemented responsive design to meet the portability requirement.

On the backend, the database is updated daily at 0am to refresh the expected progress. The commit messages are supposed to follow a format like "Task: task1; Progress: 50%; Difficulty Message: Unable to update setting; Note: ********; ……", which would be later parsed and saved into database.

10
